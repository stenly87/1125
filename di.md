# Практическое задание: Настройка DI-контейнера в Avalonia
**Тема:** Внедрение зависимостей + сервисы с бизнес-логикой

---

## 🎯 Цель задания
Научиться регистрировать в DI-контейнере сервисы с несложной логикой: работа с данными и простые вычисления.

---

## 📋 Что нужно сделать

### Часть 1. Подготовка проекта
1. Убедитесь, что установлен пакет `Microsoft.Extensions.Hosting` или минимум `Microsoft.Extensions.DependencyInjection`.
2. Создайте модель `Person`:
   ```csharp
   public class Person
   {
       public string Id { get; set; } = Guid.NewGuid().ToString();
       public string Name { get; set; }
       public string Email { get; set; }
       public int Age { get; set; }
   }
   ```

---

### Часть 2. Создайте два сервиса с логикой и зарегистрируйте их в контейнере зависимостей

#### 🔹 Сервис 1: `IPersonRepository`
**Задача:** Хранить и управлять коллекцией людей.

**Интерфейс:**
```csharp
public interface IPersonRepository
{
    IEnumerable<Person> GetAll();
    void Add(Person person);
    void Update(Person person);
    void Delete(Person person);
    Person? GetById(string id);
}
```

**Класс**
```csharp
public class PersonRepository : IPersonRepository
{
   реализация
}
```
**Требования к реализации:**
- Внутри используйте `List<Person>` как хранилище (in-memory)
- При старте добавьте 3–5 тестовых людей "по умолчанию"
- Метод `Add` должен автоматически задавать `Id`, если он пустой
- Зарегистрируйте сервис как **Singleton** (подумайте, почему это важно)

---

#### 🔹 Сервис 2: `IStatisticsService`
**Задача:** Считать простую статистику по людям.

**Интерфейс:**
```csharp
public interface IStatisticsService
{
    int GetTotalCount();
    double GetAverageAge();
    Person? GetOldestPerson();
    int GetCountByAgeRange(int minAge, int maxAge);
}
```

**Класс**
```csharp
public class StatisticsService : IStatisticsService
{
   реализация
}
```
**Требования к реализации:**
- Все методы работают с данными из `IPersonRepository`
- Если людей нет — `GetAverageAge()` возвращает 0, `GetOldestPerson()` возвращает `null`
- Подсказка: инжектите `IPersonRepository` в конструктор `StatisticsService`
- Зарегистрируйте сервис как **Singleton** или **Transient** (обоснуйте выбор в комментарии)

---

### Часть 3. Используйте сервисы во ViewModel

1. Откройте `MainWindowViewModel` (или создайте `PersonListViewModel`).
2. Добавьте в конструктор параметры:
   - `IPersonRepository repository`
   - `IStatisticsService statistics`
3. Реализуйте команды:

   **➕ `AddPersonCommand`:**
   - Создаёт нового `Person` из данных формы
   - Добавляет через репозиторий
   - Обновляет отображаемый список

   **📊 `ShowStatsCommand`:**
   - Получает статистику через `IStatisticsService`
   - Формирует строку вида:  
     `"Всего: 5 | Средний возраст: 28.4 | Самый старший: Иван (45)"`
   - Показывает через `MessageBox` или выводит в Label

   **🔄 `RefreshListCommand`:**
   - Загружает актуальный список из репозитория
   - Обновляет `ObservableCollection<Person>` для привязки к UI

4. Привяжите команды к кнопкам, а список — к `ListBox`/`DataGrid`.

---

### Часть 4. Проверка
Запустите приложение и проверьте:
- [ ] При старте загружаются тестовые люди
- [ ] Добавление нового человека работает, он появляется в списке
- [ ] Кнопка "Показать статистику" считает корректно
- [ ] После добавления/удаления статистика обновляется

## ✅ Критерии успешного выполнения

| Критерий | Да/Нет |
|----------|--------|
| Оба сервиса имеют интерфейсы и реализацию в отдельных файлах | ☐ |
| Сервисы зарегистрированы в `Program.cs` до `.Build()` | ☐ |
| ViewModel получает сервисы через конструктор (без `new`) | ☐ |
| `StatisticsService` использует `IPersonRepository` через DI | ☐ |
| Приложение не падает и реагирует на действия пользователя | ☐ |

---

## ⚠️ Частые ошибки

❌ Не создавайте сервисы через `new` внутри ViewModel — только через DI  
❌ Не делайте `new List<Person>()` внутри каждого метода репозитория — данные будут "теряться"  
❌ Не забывайте, что `ObservableCollection` нужно менять через `Add()`/`Remove()`, а не пересоздавать  
❌ Не регистрируйте сервисы после `.Build()` — контейнер уже не примет новые регистрации  
❌ Не возвращайте `null` из `GetAll()` — лучше `Enumerable.Empty<Person>()`

## 💡 Подсказки для тех, кто застрял

> 🔹 Если `StatisticsService` не видит данные репозитория — убедитесь, что оба сервиса зарегистрированы как Singleton (или что репозиторий — Singleton)  
> 🔹 Если список не обновляется в UI — проверьте, что вы используете `ObservableCollection<T>` и меняете его методами, а не присваиваете новую коллекцию  
> 🔹 Если `GetAverageAge()` делит на ноль — добавьте проверку `if (people.Any())` перед вычислениями  
> 🔹 Если не инжектится репозиторий в статистику — зарегистрируйте оба сервиса в контейнере, DI сам разрешит зависимость
