# Лабораторная работа: Разработка системы тестирования "Equestria Knowledge"

**Цель:** Разработать десктопное приложение на C# с использованием фреймворка **Avalonia UI** и базы данных **MySQL**. Приложение должно позволять пользователю проходить тесты, подсчитывать результат и сохранять историю прохождений.

---

## 1. База Данных

* База данных уже разработана, создавать новую не нужно. Данные для подключения:
* Сервер `ё192.168.200.13`
* Название `pony_test_db`
* Пользователь `student`
* Пароль `student`

### Схема таблиц

1.  **Таблица `Tests`** (Список доступных тестов)
    *   `id` (INT, PK, Auto Increment)
    *   `title` (VARCHAR(255)) — Название теста.

2.  **Таблица `Questions`** (Вопросы тестов)
    *   `id` (INT, PK, Auto Increment)
    *   `test_id` (INT, FK -> Tests.id) — Привязка к тесту.
    *   `question_text` (TEXT) — Текст вопроса.
    *   `option_a` (VARCHAR(255)) — Вариант ответа А.
    *   `option_b` (VARCHAR(255)) — Вариант ответа Б.
    *   `option_c` (VARCHAR(255)) — Вариант ответа В.
    *   `correct_option` (CHAR(1)) — Правильный ответ ('A', 'B' или 'C').

3.  **Таблица `Results`** (История результатов)
    *   `id` (INT, PK, Auto Increment)
    *   `user_name` (VARCHAR(100)) — Имя пользователя.
    *   `test_id` (INT, FK -> Tests.id) — Какой тест проходил.
    *   `score_percent` (DECIMAL(5,2)) — Процент правильных ответов.
    *   `grade` (VARCHAR(2)) — Итоговая оценка ("2", "3", "4", "5").
    *   `test_date` (DATETIME) — Дата и время прохождения.

## 2. Функциональные требования к приложению

Приложение должно состоять из трех окон (Views).

### Окно 1: Авторизация и выбор теста (`MainWindow`)
*   **Поле ввода:** `TextBox` для ввода имени студента (обязательное поле).
*   **Выбор теста:** `ComboBox` или `ListBox`, который при загрузке окна заполняется названиями тестов из таблицы `Tests`.
*   **Кнопка:** "Начать тестирование".
*   **Логика:** При нажатии проверяется имя. Если имя введено и тест выбран, открывается Окно 2. Имя и ID выбранного теста передаются в следующее окно.

### Окно 2: Прохождение теста (`TestWindow`)
*   **Отображение:** Показывает текущий вопрос (текст) и 3 варианта ответа (`RadioButton`).
*   **Навигация:** Кнопки "Назад" и "Вперед".
*   **Логика:**
    *   При загрузке окна все 20 вопросов выбранного теста загружаются в список.
    *   Пользователь выбирает вариант ответа.
    *   Кнопка "Вперед" на последнем вопросе меняется на "Завершить".
    *   При нажатии "Завершить" подсчитывается количество правильных ответов и открывается Окно 3.

### Окно 3: Результат (`ResultWindow`)
*   **Отображение:**
    *   Имя пользователя.
    *   Название теста.
    *   Количество правильных ответов (например, 18 из 20).
    *   Процент успеха.
    *   **Оценка:** Рассчитывается автоматически по формуле:
        *   90% - 100% → **5**
        *   70% - 89% → **4**
        *   50% - 69% → **3**
        *   0% - 49% → **2**
*   **Сохранение:** При открытии этого окна (или по кнопке "Сохранить") данные записываются в таблицу `Results` в БД.
*   **Кнопка:** "В главное меню" (закрывает текущие окна и возвращает к Окну 1).

1.  **Стек:** C#, Avalonia UI, MySqlConnector.
2.  **Архитектура:** Паттерн **MVVM**, DI-контейнер для регистрации сервисов.
3.  **Подключение к БД:** Строку подключения (`Connection String`) вынести в конфигурационный файл, получать через DI.
4.  **Обработка ошибок:** Предусмотреть `try-catch` блоки для обработки ошибок подключения к БД.
